Explain below how all the operators (mergeMap, concatMap, switchMap, exhaustMap) work in the context of the example provided.

Example overview

The outer observable emits values 1, 2, and 3 with a 1s delay between each emission. For every emitted value the inner observable emits the value multiplied by 10 after a 2s delay.

mergeMap (concurrent)

mergeMap subscribes to each inner observable as the outer emits. Multiple inner observables can run simultaneously and their results are merged into the output stream.

  • Outer emits 1 â†’ inner emits 10 after 2s.
  • If the outer emits 2 before the first inner completes, mergeMap also subscribes to the inner for 2.
  • Both inner observables run concurrently; their emissions interleave in the output.

concatMap (sequential)

concatMap queues inner observables and waits for each to complete before subscribing to the next, preserving the order of outer emissions.

switchMap (latest only)

switchMap cancels any previous inner observable when a new outer value arrives, keeping only the latest inner active.

exhaustMap (ignore while busy)

exhaustMap ignores new outer emissions while an inner observable is still active. It only processes the next outer value after the current inner completes.

Summary

  • mergeMap: concurrent processing (multiple active inners).
  • concatMap: sequential processing (ordered, one at a time).
  • switchMap: keeps only the latest inner (cancels previous).
  • exhaustMap: ignores new values while busy (one inner at a time, skips others).
